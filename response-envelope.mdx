Single resources are wrapped in `data`. List endpoints additionally return cursor-pagination metadata. Errors are returned with a stable `error_code`.

<CodeGroup>

```json Single
{
  "data": { /* resource fields */ }
}
```

```json List
{
  "data": [ /* items */ ],
  "meta": {
    "count": 30,
    "has_more": true,
    "next_cursor": "14d794d4-7281-465d-afe1-fc0ff5cdcf91"
  }
}
```

```json Error
{
  "error": "insufficient_scope",
  "error_code": "INSUFFICIENT_SCOPE",
  "message": "This key does not have the required 'orders:write' scope.",
  "required": "orders:write",
  "request_id": "6ba7b810-9dad-11d1-80b4-00c04fd430c8"
}
```

</CodeGroup>

## Pagination

List endpoints are cursor-paginated. Pass `limit` (max 100) and follow `meta.next_cursor` via the `after` query param to fetch the next page. Ordering is stable (`id ASC`) so concurrent inserts don't cause skips or duplicates.
